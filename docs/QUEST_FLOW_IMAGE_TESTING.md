# Как протестировать преобразование изображений (Фаза 1)

Три уровня проверки — от самого быстрого (без Telegram) к полному (через Quest Flow).

---

## 0. Подготовка (один раз)

1. **Перезапустить backend** — на старте идемпотентно накатятся 2 новые колонки
   (`questflow_image_model`, `questflow_image_presets`). Ничего не дропается.
   - локально: перезапустить `npm run dev`;
   - на VPS: `git pull && npm run build && pm2 restart …`.
2. Аккаунт должен быть на тарифе **Enterprise** (иначе 402).
3. Настройки → **Gemini API**: задать и проверить ключ («Сохранить и проверить» → статус *active*).
4. Там же ниже — **«Модель генерации изображений»**: выбрать модель.
   - для правки/композиции/коллажа берите **Nano Banana** или **Nano Banana Pro** (не Imagen);
   - Imagen — только «Генерация с нуля».
5. (Для UI лучше тестировать в интерфейсе на **русском или английском** — остальные языки
   получат англ. фолбэк до прогона `npm run i18n:propagate`.)

---

## 1. Быстрый тест — кнопка «Тест» в настройках (без Telegram) ⭐

Самый быстрый способ убедиться, что генерация работает и промт даёт нужный результат.

1. Настройки → **Quest Flow** → «Блоки обработки изображений» → **«Добавить блок»**.
2. Заполнить блок:
   - **Функция** — выбрать из списка;
   - **Дополнение к промту** — если нужно (необязательно, базовый промт уже есть);
   - **Референс-картинки** — для функций «Композиция / Примерка / Перенос стиля / Брендирование /
     Консистентность персонажа» загрузить фикс. изображение (лого/товар/стиль/лицо).
3. Внизу блока — раздел **«Проверить блок»**:
   - приложить пробное фото (если функция требует фото клиента);
   - при желании вписать текст запроса;
   - нажать **«Тест»**.
4. Через ~5–15 c появится **сгенерированная картинка** прямо под блоком. Под ней
   («Промт, отправленный в модель») — что именно ушло в Gemini.

> Тест не требует сохранения и не пишет в чат — можно итеративно крутить промт/референсы.
> Чек-лист по функциям — в конце документа.

Если вместо картинки — **красная ошибка**, см. раздел 4.

---

## 2. Тест эндпоинта напрямую (curl) — имитация Quest Flow

Проверяет ровно тот путь, который будет дёргать Quest Flow. Нужен `preset_key`
(скопировать из блока — иконка копирования рядом с «Ключ пресета») и API-ключ Quest Flow.

**Правка фото (multipart):**
```bash
curl -X POST http://localhost:3001/api/quest-flow/inbound-media \
  -H "Authorization: Bearer vbvx_qf_ВАШ_КЛЮЧ" \
  -F "telegram_bot_id=111" \
  -F "telegram_user_id=222" \
  -F "preset_key=ВАШ_preset_key" \
  -F "text=сделай фон белым" \
  -F "file=@/путь/к/photo.jpg"
```

**Генерация с нуля (JSON):**
```bash
curl -X POST http://localhost:3001/api/quest-flow/inbound \
  -H "Authorization: Bearer vbvx_qf_ВАШ_КЛЮЧ" \
  -H "Content-Type: application/json" \
  -d '{"telegram_bot_id":"111","telegram_user_id":"222","preset_key":"ВАШ_preset_key","text":"красная спортивная машина на закате"}'
```

Ожидаемый ответ:
```json
{ "ok": true, "response": { "text": "...", "imageUrl": "https://.../qf-gen-xxxx.png", "imageMime": "image/png" }, ... }
```
Открой `imageUrl` в браузере — должна показаться картинка.

> Порт замените на свой (бэкенд). На VPS — реальный домен.
> Для скачивания картинки облачным QF на сервере должен быть задан **`PUBLIC_BASE_URL`**.

## 3. Полный тест через Quest Flow → Telegram

1. В Quest Flow повесить на кнопку HTTP-блок по [docs/QUEST_FLOW_IMAGE_INTEGRATION.md](QUEST_FLOW_IMAGE_INTEGRATION.md):
   - URL `…/api/quest-flow/inbound-media` (или `/inbound`), заголовок `Authorization: Bearer …`;
   - поля `telegram_bot_id`, `telegram_user_id`, **`preset_key`**, `file`/`image`, `text`;
   - таймаут HTTP-блока **≥ 60 c**.
2. Из ответа взять `response.imageUrl` → блок **sendPhoto**, подпись = `response.text`.
3. В Telegram: нажать кнопку, прислать фото → через несколько секунд бот возвращает результат.
4. В VibeVox в чате комнаты этого клиента видно: входящее фото клиента + сгенерированная картинка.

## 4. Если что-то не так — что значит ошибка

| Симптом | Причина / что делать |
|---------|----------------------|
| `Gemini API ключ не задан…` | задать/проверить ключ в разделе Gemini API |
| 402 / `Enterprise` | аккаунт не на Enterprise |
| `Модель не вернула изображение…` | выбрана не image-модель, или сработал фильтр безопасности — смените модель/промт/фото |
| `Imagen не вернул изображение` | Imagen не умеет правку — для правки/композиции выберите Nano Banana |
| Картинка не открывается у клиента, а в тесте работает | не задан `PUBLIC_BASE_URL` на сервере |
| HTTP 400 в QF | не передан `preset_key` или обязательные `telegram_*` |
| долго и обрывается | поднять таймаут HTTP-блока в QF (≥60 c) |

Логи бэкенда содержат строки `[quest_flow/inbound]` и `[image_gen]` — там видно причину.
